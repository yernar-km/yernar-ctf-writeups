В коде есть классическая логическая ошибка (Logic Flaw), связанная с переименованием директорий и рекурсивным поиском:

Когда вы нажимаете Присоединиться (join), скрипт переименовывает директорию тир-листа, добавляя к ней ваш никнейм (например, target становится target__with_user).

Когда вы нажимаете Покинуть (leave), скрипт пытается вернуть старое имя с помощью команды mv: mv .../target__with_user .../target.

Ред-флаг скрипта: Если к моменту выхода вы уже создали свой собственный тир-лист с названием target, то target будет существовать как директория. В Linux команда mv A B (где B — существующая папка) переместит A внутрь B! То есть оригинальный тир-лист окажется внутри вашего: target/target__with_user/.

Для отображения тиров скрипт использует find "$tl_dir" -type d -name 'tier_*', не ограничивая глубину поиска (-maxdepth). Он найдет скрытые тиры внутри вложенной папки.

Проверка прав доступа (is_restricted) читает файл .meta из корня открытого тир-листа. Так как корнем теперь является ваш тир-лист target, права будут читаться из вашего .meta, где нет никаких ограничений, и вы являетесь владельцем.
